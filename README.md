# Circular-buffer

Cтруктура данных, которая использует единственный буфер фиксированного размера как будто бы он был соединен конце-to-конец.

## Преимущества:
 - Предсказуемое использование памяти - фиксированный размер
 - Высокая производительность - O(1) для добавления/удаления
 - Эффективное использование памяти - перезапись старых данных
 - Отсутствие фрагментации памяти
 - Идеально для real-time систем

## Недостатки: 
 - Фиксированный размер - нужно выбирать оптимальную емкость
 - Потеря данных при переполнении (в некоторых реализациях)
